# 星舰熔炉 · 掉落合成 × 自走棋 原型

《星舰熔炉》的可玩原型，现在包含**两个阶段的完整循环**：熔炉掉落合成（备战）→ 部署 → 自走棋自动战斗 → 结算 → 下一波。纯 JavaScript + matter.js（物理库已内置在 `lib/`，离线可用），**核心逻辑与最终 Cocos Creator (TypeScript) 版一一对应，可直接迁移**。

## 如何运行

**方式一**：直接用浏览器打开 `index.html`（双击即可）。

**方式二（推荐）**：VS Code 里装 `Live Server` 插件 → 右键 `index.html` → “Open with Live Server”，改代码自动刷新。

> 另有单文件版 `星舰熔炉_单文件可玩.html`，发别人或手机打开更方便。

## 玩法循环

1. **备战（熔炉）**：移动瞄准、点击投放零件，相同的碰一起合成升级。合成到 **5 级“攻击无人机”及以上**的战舰会带一个**阵营 + 舰种**标签（外圈颜色=阵营），顶部实时显示“当前羁绊”。堆过“过载线”约 2 秒 → 熔炉过载，本局结束。
2. **部署出战**：点「部署出战」，把熔炉里最强的最多 6 艘战舰派上战场（自动按血量排前后排）。
3. **自动战斗**：舰队按阵营/舰种**羁绊**自动开打，血条/伤害/暴击/护盾/治疗/复活实时结算。必要时点「战术技·全体齐射」放一次群体爆发。
4. **结算**：胜利得金币、进下一波；数波后是**星区 Boss**；失败可「重试本波」回熔炉重建。

## 四阵营 × 四舰种羁绊（2/4 档）

- 帝国（攻击）、异星（回血/复活）、机械（护盾/召唤）、赛博（暴击/攻速）
- 突击（前排减伤）、炮舰（增伤/溅射）、无人机（召唤僚机）、辅助（治疗）

凑齐同类 ×2 激活第一档，×4 激活更强档。数值见配套《羁绊与关卡数值表》。

## 已实现

掉落合成（11 级链）、战舰双标签、羁绊实时预览与结算、部署布阵（自动前后排）、自动战斗（寻敌/攻击/暴击/护盾/再生/治疗/复活/溅射/召唤援军）、旗舰战术技、波次 + 星区 Boss、金币、过载判定、科幻渲染与特效、战斗超时保护。

## 还没做（后续阶段）

**升星**（2 艘同名合 2★、再合 3★）、**手动拖拽布阵**（当前自动排）、**完整经济**（利息/商店/花金币招募援军刷新）、**广告变现点位**、正式美术与音效、更多阵营/舰种与更深数值。这些在 GDD v2 / PRD / 数值表里都有设计，可按 MVP 优先级继续叠。

## 想调什么改哪里（`game.js` 顶部）

- `TIERS` 每级战舰半径/颜色/名字；`VALUE` 战力
- `DROP_POOL` 投放概率；`DEPLOY_MIN` 可上阵起始等级；`FLEET_CAP` 部署上限
- `B_HP/B_ATK/B_SPD` 战斗基础属性；`FAC/CLS` 阵营与舰种
- `computeSynergy()` 羁绊数值；`genEnemies()` 敌军/ Boss 强度；`engine.gravity.y` 重力

## 迁移到抖音小游戏

抖音小游戏用 JS/TS 运行时，语言一致。落地时把 `game.js` 逻辑迁进 Cocos Creator 的 TypeScript 组件：物理从 matter.js 换成 Cocos 内置 Box2D（刚体/碰撞回调概念一致），渲染换成 Cocos 节点/精灵。合成、羁绊结算、自动战斗、Boss 这些纯逻辑几乎可原样复用。

## 下一步建议

先感受“合成→凑羁绊→自动战斗”的节奏是否有趣、羁绊是否有策略感、敌军强度松紧；确认后再加“升星”和“手动布阵”，把自走棋深度补齐，然后接广告点位与正式美术，按 PRD 的 M2→M4 推进。
