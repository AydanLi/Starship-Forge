# 星舰熔炉 · 掉落合成 × 自走棋

> **两个版本**：仓库根目录是 **web 原型**（浏览器即开即玩，快速迭代验证）；`cocos/` 是 **Cocos Creator 3.8 工程**（TypeScript，面向抖音小游戏发布，玩法逻辑与原型一致）。Cocos 版的打开/预览/抖音构建步骤见 `cocos/移植说明.md`。

《星舰熔炉》可玩原型，完整循环：**熔炉掉落合成（备战）→ 拖拽编队/升星 → 自走棋自动战斗 → 结算 → 下一波**。纯 JavaScript + matter.js（物理库已内置在 `lib/`，离线可用），核心逻辑与最终 Cocos Creator (TypeScript) 版一一对应，可直接迁移。

## 如何运行

- **直接双击** `index.html` 用浏览器打开；或
- **VS Code**：装 `Live Server` 插件 → 右键 `index.html` → “Open with Live Server”，改代码自动刷新。
- 另有单文件版 `星舰熔炉_单文件可玩.html`，发别人/手机打开更方便。

## 玩法循环

1. **备战（熔炉）**：移动瞄准、点击投放零件，相同的碰一起合成升级。合成到 **5 级“攻击无人机”及以上**的战舰带一个**阵营+舰种**标签（外圈颜色=阵营）。堆过“过载线”约 2 秒 → 熔炉过载，本局结束。
2. **编队部署**：点「编队部署」进入编队界面。**拖动战舰**布阵——前排扛伤、后排受保护输出；**把相同战舰（同级同星）叠到一起会升星**（★→★★→★★★，属性 ×1 / ×1.8 / ×3.2）。顶部实时显示阵中激活的羁绊，底部“待编区”放暂不上阵的战舰。排好点「开战」。
3. **自动战斗**：舰队按阵营/舰种羁绊自动开打，前排优先被攻击、吃“突击”减伤；暴击/护盾/再生/治疗/复活/溅射/召唤援军实时结算。必要时点「战术技·全体齐射」放群体爆发。
4. **结算**：胜利得金币、进下一波；每星区第 3 波是 **Boss**；失败可「重试本波」回熔炉重建。

## 四阵营 × 四舰种羁绊（2/4 档）

帝国（攻击）、异星（回血/复活）、机械（护盾/召唤）、赛博（暴击/攻速）；突击（前排减伤）、炮舰（增伤/溅射）、无人机（召唤僚机）、辅助（治疗）。凑同类 ×2 激活第一档、×4 更强档。数值见配套《羁绊与关卡数值表》。

> 注意升星的取舍：两艘合成一艘会**减少该羁绊的计数**（升星变强 vs 保持羁绊广度），是策略点。当前升星按“同级同星”合成，合成后继承目标的阵营/舰种。

## 已实现

掉落合成（11 级链）、战舰双标签、**拖拽编队布阵（前后排）**、**升星（★/★★/★★★）**、羁绊实时预览与结算、自动战斗（寻敌/暴击/护盾/再生/治疗/复活/溅射/召唤援军）、旗舰战术技、波次 + 星区 Boss、金币、过载判定、科幻渲染与特效、战斗超时保护、**剧情系统（序章 / 每星区进场 / Boss 登场台词 / 通关结局，剧情卡形式，点击继续）**、**经济消费（备战阶段：刷新队列 8💰 / 招募援军 15💰）**、**广告点位（模拟激励视频 3 秒）：免费援军、胜利金币双倍、败北「旗舰超载」攻击+50% 重打**。

## 经济与广告

- 金币来源：打赢波次（基础 + Boss 加成，随星区提升）。消费：**刷新队列**（换当前+下一个投放件，凑想要的合成路线）、**招募援军**（直接空降一艘可上阵战舰）。
- 广告点位均为**玩家主动点击换增益**，原型用 3 秒模拟倒计时代替真广告；正式版在 `startAd()` 处替换为抖音激励视频 SDK（`tt.createRewardedVideoAd`，`onClose` 确认看完再发奖励），三个触发点与 PRD 的点位设计一一对应。
- 数值（费用/奖励/加成幅度）集中在 `game.js` 顶部 `REFRESH_COST / RECRUIT_COST / AD_SECONDS` 与 `endBattle()`。

## 剧情

世界观：大寂灭后，你是曲率熔炉舰「星炉号」的末代舰队指挥官，从残骸中重建舰队，打穿海盗→机械→虫族→叛变AI→帝国五个星区，回到被封锁的家园「新伊甸」。完整设定（世界观、五大星区弧、Boss 背景与台词、结局与续作钩子）见 `剧情设定.md`。剧情文案集中在 `game.js` 顶部的 `INTRO / SECTORS / BOSS_TAUNT / ENDING` 常量，随时可改。

## 登录与主界面

启动后进入**登录页**：输入「指挥官代号」（2~12字，即用户ID）或点「游客进入」。每个代号拥有**独立存档**（key 按 uid 隔离），记住上次登录、下次直达主界面。正式版把 `js/user.js` 的 login 换成抖音 `tt.login`（openid 作 uid）即可，接口不变。

**主界面**：开始游戏（有进度续档/无进度播序章）、关卡选择、系统设置（音效开关/清除本账号存档/切换指挥官）。**关卡选择**是一张归途星图：从「星炉号·启航点」到顶端家园「新伊甸」，五个星区节点按剧情排布，打通才点亮，未解锁显示「航道封锁·信号未探明」；点已点亮的星区可回跳重打（进度解锁用 maxLevel 记录，不会因回跳丢失）。

## 音效

程序化合成音效（Web Audio API，零音频文件）：投放、合成（随等级变厚重，9级以上带轰鸣）、升星琶音、部署、开战警报、激光（全局节流防刺耳）、爆炸、Boss 击毁、胜利/败北、战术技、金币、拒绝、剧情卡、熔炉过载，共 15 种。右下 🔊 按钮静音（偏好会记住）。浏览器策略要求首次点击后才出声。正式版换真实音频文件时只改 `js/audio.js`，调用点不动。

## 存档与防篡改

进度（星区/波次/金币/分数/最高合成）自动存档：进入备战、战斗结算、金币变动时写入，浏览器关闭前兜底写一次。再次打开自动续档（跳过序章）；「重新开始」= 清档从头。

防篡改分两级：**现在（本地）**——存档带版本号和加盐校验和，手改存档内容会导致校验失败，游戏拒收并重置（能挡住直接编辑存档的普通玩家；盐值在客户端，防不了逆向，这是本地方案的天然上限）。**未来（服务器，接口已预留在 `js/save.js`）**——登录后拉取服务器权威存档，服务器用私钥 HMAC 签名、客户端只验签不持钥；本地校验失败即强制采用服务器档，双方有效则按进度高者合并回传。抖音端落地时本地存储换 `tt.setStorageSync`，网络走 `tt.request`，存档结构不变。

## 代码结构（模块化）

代码按「数据层→服务层→系统层→表现/交互→装配」分层拆在 `js/` 下，全局只有一个命名空间 `SF`。加载顺序 = 依赖顺序，写在 `index.html` 和 `build.py` 两处（新增模块要同时改这两处）。设计详见 `模块化设计.md`。

```
index.html            页面骨架 + 模块加载顺序
build.py              单文件版构建脚本（python build.py）
js/
  config.js           全部数值配置（改平衡先来这）        SF.C / SF.util
  story-data.js       剧情文案（改台词只改这）            SF.STORY
  state.js            运行时状态                          SF.G
  user.js             用户ID登录（★接抖音tt.login只改这）  SF.user
  save.js             本地存档/校验和/云同步接口预留       SF.save
  audio.js            程序化合成音效/静音开关              SF.audio
  fx.js               粒子/光束/飘字/屏震                 SF.fx
  synergy.js          羁绊计算（纯函数）                  SF.synergy
  fleet.js            单位工厂/敌军生成/站位              SF.fleet
  forge.js            熔炉物理/投放/合成/过载             SF.forge
  board.js            编队阵位/拖拽/升星                  SF.board
  battle.js           自动战斗/战术技/胜负                SF.battle
  economy.js          金币消费/结算页广告入口             SF.econ
  ads.js              广告点位（★接抖音SDK只改这个文件）  SF.ads
  story.js            剧情队列状态机                      SF.storySys
  menu.js             登录/主界面/星图选关/设置            SF.menu
  render.js           全部 canvas 绘制（只读状态）        SF.render
  ui.js               DOM 按钮/提示条                     SF.ui
  input.js            指针事件路由                        SF.input
  main.js             接线/流程/主循环                    SF.main
lib/matter.min.js     物理库（已内置）
```

## 想调什么改哪里

数值平衡 → `js/config.js`（TIERS/VALUE/DROP_POOL/B_HP…/STAR_MUL/费用）；羁绊效果 → `js/synergy.js`；敌军/Boss 强度 → `js/fleet.js` 的 `genEnemies()`；剧情台词 → `js/story-data.js`；广告时长与对接 → `js/ads.js`；重力手感 → `js/forge.js` 的 `engine.gravity.y`。

## 迁移到抖音小游戏

抖音小游戏用 JS/TS 运行时，语言一致。落地时每个 `js/` 模块对应一个 Cocos Creator 的 TypeScript 类/服务：`synergy/fleet` 等纯逻辑可原样复用；`forge` 的 matter.js 换成 Cocos 内置 Box2D；`render/input/ui` 被 Cocos 节点与事件系统替代；`ads.js` 换成 `tt.createRewardedVideoAd` 封装。映射表见 `模块化设计.md` 第六节。
