# 星舰熔炉 · 即梦背景音乐提示词

用途:在即梦「音频」入口生成游戏背景音乐。整体风格统一为科幻、深空、电子、史诗感,避免人声和歌词,便于在 Cocos 中循环播放。

## 通用要求

- 纯音乐,无人声,无歌词,无明显版权风格模仿。
- 科幻未来感,深空氛围,电子合成器,低频脉冲,轻微机械质感。
- 适合手游 / 小游戏循环播放,结尾自然收束,方便无缝循环。
- 音量动态不要过大,避免遮挡按钮音效、合成音效、战斗反馈。
- 推荐时长:60 到 90 秒(失败曲除外,见对应条目)。

## 1. 主界面 / 星图

```
科幻深空游戏主菜单背景音乐,纯音乐,无人声,无歌词。画面感是末代舰队在星海中重新启航,远方有被封锁的家园,新伊甸若隐若现。音乐风格为 synthwave、ambient electronic、cinematic sci-fi,慢速铺底,温暖但带孤独感。使用宽广的合成器 Pad、柔和的低频脉冲、微弱星尘音色、少量钟琴和空间回响。情绪是希望、漂泊、归途、克制的史诗感。节奏不要太强,适合作为主界面和关卡选择背景,60 到 90 秒,可循环。
```

> 登录页直接复用这首,不必单独生成。

## 2. 熔炉备战 / 掉落合成

```
科幻小游戏备战阶段背景音乐,纯音乐,无人声,无歌词。场景是曲率熔炉正在运转,玩家不断投放残骸零件,合成芯片、电池、炮台和战舰。音乐风格为轻快电子、future bass、synth arpeggio、mechanical groove。节奏中速,带稳定律动,鼓点轻,不要过于激烈。使用短促合成器琶音、机械脉冲、明亮电子音色、低频引擎声,表现紧张但上瘾的合成手感。情绪是专注、期待、逐步升级、越合越爽。适合长时间循环播放,60 到 90 秒。
```

> 编队部署阶段沿用这首(备战与布阵同属一次战前流程,不切歌)。

## 3. 普通自动战斗

```
科幻舰队自动战斗背景音乐,纯音乐,无人声,无歌词。场景是玩家布阵后的战舰舰队与敌军在星区中交火,激光、护盾、无人机、炮舰齐射。音乐风格为 cinematic electronic、synth rock、hybrid trailer。中高速节奏,鼓点清晰,低频有推进感,旋律不要太复杂。使用电子鼓、脉冲贝斯、合成器 Lead、金属打击、短促弦乐层,表现战术感和火力压迫。情绪是紧张、果断、科技感、胜负拉扯。适合作为普通波次战斗循环,60 到 90 秒。
```

## 4. Boss 战

```
科幻星区 Boss 战背景音乐,纯音乐,无人声,无歌词。场景是舰队迎战巨大 Boss 母舰、机械泰坦、虫族女皇、叛变 AI 或帝国歼星舰。音乐风格为 epic sci-fi battle、dark synth、orchestral hybrid、trailer music。节奏强烈,低频厚重,鼓组宏大,加入压迫性的铜管或合成铜管、工业打击、警报式合成器音色。情绪是危险、庞大、决战、绝地反击。中段要有能量逐渐上升的段落,适合释放全体齐射和 Boss 爆炸。不要恐怖过头,要保持热血和爽感,60 到 90 秒,可循环。
```

## 5. 胜利 / 结算

```
科幻游戏通关奖励背景音乐,纯音乐,无人声,无歌词。场景是舰队完成星区任务后穿过星尘返航,远处星门打开,奖励结算界面亮起温暖光芒。音乐风格为 triumphant synthwave、cinematic electronic、short victory loop。节奏中速偏轻快,旋律明亮,使用上升合成器、温暖 Pad、清脆电子打击和少量史诗鼓点。情绪是胜利、奖励、希望、继续前进。适合作为胜利结算和金币奖励页面背景,30 到 60 秒,可循环或自然结束。
```

## 6. 失败 / 熔炉过载

> ⚠️ 初版措辞(含「失败结算/警报/败退」)被即梦审核拦截,以下为已过审措辞。

```
科幻游戏低落情绪短曲,纯音乐,无人声,无歌词。场景是星舰引擎缓缓熄火,舱内灯光暗下来,舰队暂时返航休整,准备下一次出发。音乐风格为 dark ambient electronic、cinematic sting。慢速,低频缓缓下沉,合成器音色由强渐弱,少量金属残响和引擎减速的质感。情绪是遗憾、沉静,但不绝望,结尾留一丝微弱上扬,暗示重新出发。15到30秒,单次播放,不需要循环。
```

## 7. 剧情卡 / 序章

> ⚠️ 初版措辞(含「大寂灭/被封锁/独白」)被即梦审核拦截,以下为已过审措辞。

```
科幻游戏宁静氛围背景音乐,纯音乐,无人声,无歌词。场景是深空星海缓缓流动,一张古老的星图徐徐展开,旅人凝望远方的家园。音乐风格为 space ambient、drone、minimal electronic。极慢速,无鼓点或只有极轻微的脉冲,使用宽广的合成器 Pad、稀疏的钟琴单音、微弱的星尘电波质感,像宇宙深处传来的信号。情绪是苍凉、神秘、宁静而有史诗感。整体音量小、动态平缓,适合作为文字页面的衬底。60到90秒,可循环。
```

## 8. 通关结局 / 回到新伊甸

```
科幻游戏通关结局音乐,纯音乐,无人声,无歌词。场景是舰队穿过最后一道星门,家园新伊甸的封锁解除,恒星的光洒进舰桥,漫长归途在此刻结束。音乐风格为 emotional synthwave、cinematic orchestral hybrid。前段安静,用音色纯净的钢琴或柔和合成器独奏铺垫;中段弦乐层、温暖 Pad 与史诗鼓点逐渐加入,能量稳步上升;结尾宏大而温暖地收束,不要戛然而止。情绪是回家、释然、荣耀、对牺牲者的告别。60 到 90 秒,单次播放为主,结尾自然收束。
```

## 9. 星区战斗氛围变体(可选第三批)

普通战斗只用第 3 首也完全够用。以下五首是**按星区换战斗 BGM** 的进阶方案,与美术的五星区背景/杂兵/Boss 切换呼应,后期资源包体允许时再生成。每首都以第 3 首的节奏骨架为基础,只换配器与氛围,保证切星区时听感连贯。

### 9.1 星区一 · 残骸带(海盗)

```
科幻舰队战斗背景音乐,纯音乐,无人声,无歌词。场景是残骸带里与太空海盗交火,拼凑的掠夺舰横冲直撞。音乐风格为 synth rock、electronic hybrid,在电子鼓和脉冲贝斯的基础上加入粗粝的电吉他质感、俏皮而挑衅的合成器 riff。中高速节奏,情绪是野性、无法无天、以战养战的痞气。保持科幻底色,不要变成纯摇滚。60 到 90 秒,可循环。
```

### 9.2 星区二 · 机械坟场(机械军团)

```
科幻舰队战斗背景音乐,纯音乐,无人声,无歌词。场景是废弃的机械坟场,沉睡的机械哨兵与铁卫列队苏醒,精确而冷酷地推进。音乐风格为 industrial electronic、mechanical percussion。中高速节奏,使用大量金属敲击、齿轮与液压质感的打击乐、冰冷规整的合成器脉冲,旋律极简,像机器军团的行军。情绪是冷酷、压迫、精密、无情。60 到 90 秒,可循环。
```

### 9.3 星区三 · 虫巢(虫族)

```
科幻舰队战斗背景音乐,纯音乐,无人声,无歌词。场景是深入虫巢星区,生物舰群如潮水般涌来,女皇在巢穴深处苏醒。音乐风格为 dark hybrid electronic、organic sound design。中高速节奏,在电子鼓底上叠加湿润的生物质感音效、窸窣爬行般的高频打击、不和谐的滑音弦乐涌动。情绪是密集、蠕动、本能的威胁,略带恶心但绝不恐怖过头,保持爽快的战斗推进感。60 到 90 秒,可循环。
```

### 9.4 星区四 · 深渊(叛变 AI)

```
科幻舰队战斗背景音乐,纯音乐,无人声,无歌词。场景是深渊星区,叛变 AI 的几何构造体从数字虚空中浮现,零号意志冷冷注视。音乐风格为 glitch electronic、dark minimal techno。中高速节奏,使用故障音效、比特破碎质感、精确到冷漠的合成器序列、深不见底的 Sub Bass。旋律稀少,情绪是诡异、失真、被无限算力凝视的寒意,同时保持清晰的战斗律动。60 到 90 秒,可循环。
```

### 9.5 星区五 · 帝国终末(帝国舰队)

```
科幻舰队战斗背景音乐,纯音乐,无人声,无歌词。场景是帝国终末星区,与万王之王的皇家歼星舰队展开最后的大决战,家园就在战线之后。音乐风格为 epic orchestral hybrid、imperial trailer music。中高速节奏,加入庄严的铜管齐奏、恢弘的战争鼓组、合唱式合成器 Pad(不要真人声),底层保留电子脉冲贝斯。情绪是王朝末日的庄严、宿命对决、走向终章的悲壮与热血。60 到 90 秒,可循环。
```

## 推荐生成顺序与批次

| 批次 | 曲目 | 说明 |
|---|---|---|
| 第一批(核心 5 首) | 1 主界面 → 2 熔炉备战 → 3 普通战斗 → 4 Boss 战 → 5 胜利 | 有这 5 首游戏就完整了 |
| 第二批(补全 3 首) | 6 失败 → 7 剧情卡 → 8 通关结局 | 提升情绪完整度 |
| 第三批(可选 5 首) | 9.1~9.5 星区战斗变体 | 替换第 3 首按星区切换,注意包体大小 |

如果即梦一次只能生成一首,优先从主界面开始,确定整体音色后再做其它场景;后续每首生成时可在提示词里补一句「与之前生成的主界面音乐风格统一」。

> **过审经验**:即梦音乐提示词里避免出现「失败/结算/败退/警报/大寂灭/封锁/独白」这类词,容易触发「音频可能包含不适当内容」拦截;改成中性的场景描述(返航休整、星图展开等)即可通过。被拦截的请求不扣积分。

## 文件命名约定(接入 Cocos 用)

生成的音频统一转成 mp3(44.1kHz,128kbps 足够),放入 `cocos/assets/resources/audio/`,按下表命名,后续 `core/audio.ts` 接 AudioSource 时直接按 key 取用:

| 文件名 | 场景 | 播放方式 |
|---|---|---|
| `bgm_menu.mp3` | 登录 / 主界面 / 星图 | 循环 |
| `bgm_forge.mp3` | 熔炉备战 / 编队部署 | 循环 |
| `bgm_battle.mp3` | 普通自动战斗 | 循环 |
| `bgm_boss.mp3` | Boss 战 | 循环 |
| `bgm_win.mp3` | 胜利 / 结算 | 循环或单次 |
| `bgm_lose.mp3` | 失败 / 熔炉过载 | 单次 |
| `bgm_story.mp3` | 剧情卡 / 序章 | 循环 |
| `bgm_ending.mp3` | 通关结局 | 单次 |
| `bgm_battle_s1.mp3` ~ `bgm_battle_s5.mp3` | 星区战斗变体(可选) | 循环,存在则优先于 `bgm_battle` |

> 抖音小游戏主包限制 4MB、总包 20MB,当前 tex 已约 5MB;13 首 60~90 秒 128kbps mp3 约 12~15MB,建议音频放**分包或远程资源**,或先只带核心 5 首(约 5MB)。
